9/5/2025 -2.4 - Minor update
 -Fixed depreciation warnings.
 -FastROM addressing now uses "|!bank" rather than "!F". This makes it standard for sa-1 hybrids.
 -Macro define subroutine name update:
 --Instead of first offsetting the address then set the define at that offsetted location, it now
   set the define intuitively at the current location, then offset for the next define to set to.
   This also makes it no longer necessary for the first JML define to be manually set via
   "!JMLListStart".
 --Renamed macro "SetDefine" to "SetSharedSubDefine". Not a safe idea to have generic names that
   cannot be re-defined or overwritten.
 -Added a failsafe for potential macro redefinition warning/errors, due to a bug that asar's
  includeonce isn't foolproof when two ASM files incsrcs to the same ASM file with different
  paths. See this: https://github.com/RPGHacker/asar/issues/287
 -Modernized the readme file. Dark background and white text, along with making code selection
  easy to copy via CTRL+double click.
 -Readme/RoutinePorting.html includes on how to convert ASM tools' routines to a format that works
  on shared subroutines (tools' routines are often macro-based and uses macrolabels to preventing
  labels from leaking).
 -Added note on that subroutines included may be out of date.
 -Better explained why a crash happens if you apply Shared Subroutines, a 3rd-party patch that
  needs freespace, then Shared Subroutines with a longer JML list.
 -Added a JavaScript HTML file that automatically handles the JML and macro define list, along
  with detecting reused names and generating placeholder to reserve space to prevent other
  patches from occupying that area.
 -Renamed "!FreespaceU" to "!Freespace_SharedSub_JMLList" because again, generic names have a
  higher risk of it conflicting with other 3rd party ASM.
