Note: This does not handle multi-line operator or single-line operator. Each line must contain up to one item.

This tool takes a given list of subroutine names (in any of the 3 formats, see below), and convert them into shared subroutines format. Features include:

  • Generating both labels and defines in the same order (to prevent incorrect define assign to the wrong JML).
  • Scan for label/define duplicate/reuse.
  • Generate placeholder JML items and defines to prevent 3rd party patches from occupying a area of memory that will expectedly be used for the JML list.

Accepted formats are:
Just the names JML list format define list format 
SubroutineName1
SubroutineName2
SubroutineName3
 
autoclean JML SubroutineName1
autoclean JML SubroutineName2
autoclean JML SubroutineName3
 
%SetSharedSubDefine(SubroutineName1)
%SetSharedSubDefine(SubroutineName2)
%SetSharedSubDefine(SubroutineName3)

!Freespace_SharedSub_JMLList = $ (NOTE: Does not account for bank borders or memory regions)

Routines having the same name detected! This works fine if two calls point to the same routine, but may cause issues with the defines as they can conflict (will be the value of what's last sets them). The exception to this are placeholders, which you won't be using.
